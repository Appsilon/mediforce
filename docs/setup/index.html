Repository

Which Mediforce repo to deploy.

Your server needs read access to this private repo to pull code. During setup, the script will generate an SSH key on the server and pause — you’ll copy that key to your repo’s deploy keys on GitHub. The script will walk you through it step by step.

Server

Create a server or enter the IP of an existing one.

Hetzner Cloud (recommended)
  1. Open console.hetzner.cloud → your project
  2. Click Add Server
  3. Location: pick close to your users (e.g. Falkenstein or Helsinki for Europe)
  4. Image: Ubuntu 22.04
  5. Type: CPX31 (4 vCPU, 8 GB RAM)
  6. SSH keys: add yours (and any team members who need access — all keys added here get root + deploy access)
  7. Create & copy the IP address

⚠ You can add more SSH keys later via ssh-copy-id, but it’s easiest to add them all during server creation.

DNS setup

Add an A record pointing your domain to at your DNS provider. Make sure no CDN/proxy sits in front — Caddy needs direct access for TLS.

Browsers will show a security warning with a self-signed certificate. Fine for testing, not recommended for production.

Sign-in

Authentication runs on NextAuth (ADR-0002) — users live in Postgres, there is no Firebase project.

Create a Google OAuth client
  1. Open Google Cloud Console → Credentials
  2. Create credentials → OAuth client ID → Web application
  3. Authorised JavaScript origin: https://your-domain
  4. Authorised redirect URI (exactly, including the path): https://your-domain/api/auth/callback/google
  5. Copy the client ID and client secret below
Comma-separated. Enforced on every OAuth sign-in (ADR-0002 §4a).
Leave the domain list empty and any Google account on earth could sign in. The server refuses to boot in that state, so it is required here.
