Security model

Your data stays yours. Your agents stay scoped.

Pharma data is sensitive. AI agents are powerful. Mediforce gives you granular control over what each agent can access, where your data lives, and what models process it.

Isolate

Every agent runs in a sandbox. Every step has a scope.

We hear from pharma teams that they only give AI access to metadata — never real patient data. Mediforce is built for that reality. You control exactly what each agent sees, down to the table level.

Per-step data scoping

Each workflow step defines exactly which data sources, MCP servers, and tools the agent can access. Working on Study ABC-123? That agent cannot reach Study DEF-456.

Container sandboxing

Each agent runs in an isolated Docker container. Memory limits, CPU limits, read-only mounts. Container destroyed after execution.

Per-step tool sets

Different MCP servers, different tools, different environment variables per step. An agent can only use what you configure.

Namespace isolation

Workflow secrets scoped per namespace. Members verified before access. Cross-namespace access denied by default.

Access

Who can do what. Enforced, not advised.

Role-based access control at every level: namespace, workflow, step.

Step-level RBAC

Each step specifies allowed roles. The platform checks before the step can advance. Wrong role? Blocked, logged, audited. Not a warning — a hard stop.

Clinical Data Manager can approve Step 2 (data review). Cannot approve Step 5 (medical review). Medical Monitor can approve Step 5. Cannot modify Step 2. Enforced in workflow engine before any state change.

No shared accounts

Every user has a unique identity. Every action tied to a specific person. No "team logins" or generic service accounts for human actions.

Access denied = audited

Every unauthorized access attempt is logged with full context: who tried, which step, which roles they have, which were required.

Encrypt

Secrets encrypted. Audit trail tamper-evident.

Data at rest and in transit protected with industry-standard cryptography.

Workflow secrets

API keys, tokens, credentials — encrypted at rest with AES-256-GCM. Injected into agent containers at runtime, never stored in workflow definitions or logs.

Secrets stored as iv:ciphertext:authTag in Firestore. Template syntax {{SECRET_NAME}} resolved at execution time. Pre-flight validation catches missing secrets before workflow starts.

Tamper-evident audit trail

Server-written timestamps — not client clock. Append-only — no updates, no deletes. If someone tampers with an entry, the chain breaks.

Deploy

Run it on your infrastructure. Use your models.

You choose where the platform runs, where data lives, and which LLMs process it.

Self-hosted models

Run LLMs on your own infrastructure — isolated from the internet, inside your network perimeter. No data leaves your environment. Configure any model that speaks the OpenAI-compatible API as a provider.

Deploy open models (Llama, Mistral, Qwen, GLM) on your GPU cluster or private cloud. Point Mediforce at your endpoint. The platform doesn't care which model — it cares about the process around it.

Air-gapped steps

Configure individual steps to run without internet access. Network blocked at container level. Data stays inside.

Open source

Inspect every line. No vendor lock-in. Deploy on your cloud, your on-prem, your air-gapped network.

Model-agnostic

Use Claude, GPT, Gemini, or self-hosted models. Switch per step. The validation layer works the same regardless.

Security is one part of the picture.

Security controls keep data safe. Validation controls make sure AI results are trustworthy. Together, they let you deploy AI agents in regulated pharma with confidence.
