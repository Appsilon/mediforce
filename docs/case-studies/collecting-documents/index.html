The Challenge

Collecting submission documents is a manual chase.
One person tracks it all in their head.

Every study pulls documents from multiple stakeholder teams on different schedules. Keeping track of who was asked, who replied, and what still needs fixing doesn't scale past a handful of studies.

Manual outreach to every stakeholder team

The submission coordinator hand-writes and sends individual requests to each team responsible for a piece of the package, then has to remember who was asked for what and by when.

No visibility into who's actually submitted

Files land in a shared drive with no notification. The coordinator has to keep checking manually to see what's arrived and who's still missing - or wait for someone to ask.

Wrong files slip through unnoticed

A mislabeled or mismatched file - the wrong drug name on a template, for instance - can sit in the shared drive until someone happens to open it, costing days when it's caught late.

Reminders and follow-ups fall on one person

As the deadline approaches, chasing slow responders and drafting clarification requests for issues found becomes a repetitive job with no system tracking who still needs a nudge.

How It Works

One request goes out. Every file is checked. Every follow-up writes itself.

An autonomous workflow runs the collection loop end to end, and surfaces only the decisions that need the coordinator's judgment.

Human

Define stakeholders, deadlines & file specs

The submission coordinator provides one JSON file: which teams are responsible for which documents, their deadlines, and any study-specific requirements. It's the only manual data entry in the whole workflow.

Script

Fill the request email from a shared template

A markdown email template - held in the same repository as the workflow - has placeholders for study details, file specifications, and deadlines. The workflow fills them from the JSON values automatically.

Human

Preview, approve, and send

Before anything goes out, the workflow sends the coordinator a preview of the exact email, then a summary of who it'll reach. One approval, and it's sent to every stakeholder team at once.

Agent

AI agent watches the shared drive

Once the request is out, an AI agent periodically checks the SFTP / shared drive for incoming files, analyzes what's been submitted, and prepares a report - present or missing, and whether the content looks right.

Human

Coordinator decides: wait, remind, or request a fix

Reading the agent's report, the coordinator picks one of three outcomes: give stakeholders more time, trigger a deadline reminder, or flag a file that needs correcting - for example, a document filed under the wrong drug label.

Agent

Agent drafts and sends the follow-up

On remind or request-fix, an AI agent drafts a scoped, factual email - a deadline nudge or a clarification request naming the exact issue - and sends it. The loop repeats until every file is in and correct, then the workflow closes with a cost and completion summary.
