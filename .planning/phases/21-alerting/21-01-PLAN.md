  <action>
    **Per user decision**: Audit entry per failed attempt (full retry timeline). Successful sync also gets one audit entry. Boot-time eager sync: audit on failure, no webhook.

    The sync functions (`syncWithRetry`, `syncFromOpenRouter`) live in platform-infra and do NOT have access to the audit repo or CallerIdentity. The cleanest integration is: add an `onAttemptFail` callback to `syncWithRetry`, then emit audits at the call sites.

    1. **Modify `syncWithRetry` in `openrouter-sync.ts`**:
       - Add optional callback to opts: `onAttemptFail?: (attempt: number, error: Error) => void | Promise<void>`
       - In the catch block, after the console.log, call `await opts.onAttemptFail?.(attempt, lastError!)` before the sleep.
       - Keep the function's return type, error behavior, and existing signature backward-compatible (callback is optional).

    2. **Modify cron route** (`platform-ui/src/app/api/cron/model-sync/route.ts`):
       - Destructure `auditRepo` from `getPlatformServices()` alongside `modelRegistryRepo`.
       - Import `emitAudit` from `@mediforce/platform-api` and `CallerIdentity` from its auth module.
       - Create a system caller: `const systemCaller: CallerIdentity = { kind: 'apiKey' as const }` (apiKey callers map to system actor via auditActorFrom).
       - Pass `onAttemptFail` to `syncWithRetry`:
         ```typescript
         onAttemptFail: (attempt, error) => emitAudit(auditRepo, systemCaller, {
           action: 'model_sync.attempt_failed',
           description: `Sync attempt ${attempt} failed: ${error.message}`,
           entityType: 'model_registry',
           entityId: 'openrouter',
           basis: 'cron-0-3-daily',
           inputSnapshot: { attempt },
           outputSnapshot: { error: error.message },
         })
         ```
       - On success (syncWithRetry returns), emit:
         ```typescript
         emitAudit(auditRepo, systemCaller, {
           action: 'model_sync.completed',
           description: `Sync completed: ${result.synced} synced, ${result.retired} retired, ${result.reinstated} reinstated`,
           entityType: 'model_registry',
           entityId: 'openrouter',
           basis: 'cron-0-3-daily',
           inputSnapshot: {},
           outputSnapshot: { ...result },
         })
         ```
       - On final failure (catch block), emit:
         ```typescript
         emitAudit(auditRepo, systemCaller, {
           action: 'model_sync.failed',
           description: `Sync failed after all retries: ${err.message}`,
           entityType: 'model_registry',
           entityId: 'openrouter',
           basis: 'cron-0-3-daily',
           inputSnapshot: {},
           outputSnapshot: { error: err.message },
         })
         ```

    3. **Modify `eagerSyncIfStale` in `eager-sync.ts`**:
       - Add optional second parameter: `opts?: { auditRepo?: AuditRepository }` (keeps existing callers working with no changes).
       - On failure (catch block where `message` is already captured), if `opts?.auditRepo` is provided, call:
         ```typescript
         try {
           await opts.auditRepo.append({
             actorId: 'system', actorType: 'system', actorRole: 'system',
             action: 'model_sync.eager_failed',
             description: `Boot-time sync failed: ${message}`,
             timestamp: new Date().toISOString(),
             entityType: 'model_registry', entityId: 'openrouter',
             basis: 'boot-eager-sync',
             inputSnapshot: {}, outputSnapshot: { error: message },
           });
         } catch { /* audit failure must never block boot */ }
         ```
       - Use `auditRepo.append` directly (not `emitAudit`) to avoid importing platform-api into platform-infra (maintains dependency direction: infra does not depend on api).
       - Do NOT pass auditRepo from migrate-with-sync.ts -- the migrate container's single-shot nature means container logs cover failures, and the next cron window will retry + audit. Prefer simplicity.
  </action>
