# PLAN-0011 — Implementation plan for ADR-0011

Companion to [`ADR-0011-unified-canvas-first-workflow-designer.md`](./ADR-0011-unified-canvas-first-workflow-designer.md), specifically the Phase 4 AI Assistant capability wiring and the "AI Assistant — reuse from the Co-Work Workflow Designer Workflow" section.

## Steps

- **Tool schema**: Zod schema for canvas-mutation tool calls — `add_step`, `update_step`, `remove_step`, `add_transition` (snake_case tool names; mapped internally to the existing `addStep`/`updateStep`/`removeStep`/`addTransition` functions already in `workflow-editor-canvas.tsx`).
- **System prompt**: adapted from `workflow-designer.wd.json`'s `design` step prompt + `getWorkflowAuthorableJsonSchema()` + a few `docs/workflow-examples/` entries for few-shot grounding.
- **Headless handler + contract**: one handler (e.g. `assistant.ask`) — takes `{messages, model, workflowDefinition}`, calls `callOpenRouter()` with the prompt + tool schema, returns either a text reply or a tool call. No engine, no session persistence yet.
- **Route + CLI**: Next.js route adapter wiring the handler, plus `mediforce.assistant.ask()` client method and CLI command (AGENTS.md §4, dogfood the CLI).
- **Model config**: drop `ModelPicker` into the AI pane header, default to the existing `DEFAULT_MODEL` convention (`anthropic/claude-sonnet-4`).
- **Chat UI**: swap the "To be implemented" placeholder for the reused `ChatCoworkView` pieces (message list, bubble, input) — local component state only, no artifact panel, no polling.
- **Wire tool calls → canvas**: when the assistant returns a tool call, invoke the existing `addStep`/`updateStep`/`removeStep`/`addTransition` functions directly — no new mutation path.
- **L3 test**: one API-level test for the handler — given a prompt (mocked LLM response), assert it either produces the right step mutation or a plain-text reply.
- **Manual verification**: describe a workflow → it builds steps; ask a question → plain answer; ambiguous request → clarifying question back.

Each step is independently shippable and testable before moving to the next.
