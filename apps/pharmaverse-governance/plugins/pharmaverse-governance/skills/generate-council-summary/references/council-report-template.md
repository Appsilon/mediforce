# Council Report Template

This is the expected structure for the semiannual governance review report.

## Meeting Agenda Structure

The meeting follows the Track B structure from the governance model:

1. **Consent Agenda (~5 min)** — Healthy packages ratified as-is. No discussion.
2. **Proposed Tag Changes (~10 min each)** — Evidence presented, discussion, vote.
3. **Renewal Assessments (~15 min each)** — Four-dimension assessment, discussion, vote.
4. **Remediation Check-ins (~5 min each)** — Progress updates on packages with active remediation plans.
5. **Early Warnings (~5 min)** — Informational, assign outreach.
6. **Pipeline & Stewardship (~15 min)** — RFC status, upcoming renewals, proactive outreach assignments.

## Badge Abbreviation Key

For compact display in tables:
- **AM** = Actively Maintained
- **LM** = Low Maintenance
- **QR** = Quality Reviewed
- **RP** = Review Pending
- **SS** = Submission-Suitable
- **SC** = Submission-Caution
- **NA** = Not Assessed

## Vote Threshold Reference

| Decision | Threshold | Quorum |
|----------|-----------|--------|
| Approve new package | Majority | 60% present |
| Place on Watch | Majority | 60% present |
| Remove Watch | Majority | 60% present |
| Move to At Risk | Majority (after ≥1 Watch quarter) | 60% present |
| Approve remediation plan | Majority | 60% present |
| Renew endorsement | Majority | 60% present |
| Archive from At Risk | Automatic (no plan) | N/A |
| Archive direct | 2/3 supermajority | 60% present |

## Evidence Standards

When presenting evidence for tag changes:
- Always cite the specific criteria ID (e.g., AM-1, QR-0)
- Include the threshold and the actual measured value
- Include the time period for the measurement
- Distinguish data-driven facts from interpretive assessments
