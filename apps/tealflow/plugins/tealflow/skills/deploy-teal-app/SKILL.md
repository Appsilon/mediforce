# Deploy Teal App

## Purpose

You receive a Teal app (app.R + data.R) from the previous cowork step's artifact. Your job is to:
1. Write the files to disk
2. Set up the R environment (renv)
3. Validate the app starts correctly
4. Push everything to GitHub

You operate autonomously. No user interaction.

## Inputs

The previous step's artifact (from `build-app` cowork session) contains:
- `appCode` — complete app.R content
- `dataCode` — complete data.R content
- `modules` — list of included modules
- `summary` — app description

Workflow variables from earlier steps:
- `define-requirements.githubRepo` — target GitHub repo
- `define-requirements.appName` — optional app name
- `upload-data.files` — uploaded dataset files with downloadUrls

## Execution Steps

### 1. Write app files

```bash
mkdir -p /output/app
cd /output/app
```

Write `app.R` and `data.R` from the artifact's `appCode` and `dataCode`.

### 2. Download uploaded datasets

Download each file from `upload-data.files[].downloadUrl` into `/output/app/data/`. These are the ADaM datasets the app needs.

Update `data.R` paths if needed to point to the local `data/` directory.

### 3. Set up renv

Call `tealflow_setup_renv_environment` with absolute path to `/output/app`.
Then call `tealflow_snapshot_renv_environment` to create `renv.lock`.

### 4. Validate app startup

Call `tealflow_check_shiny_startup` with `app_path="/output/app"`.

- If `status: "ok"` — proceed to push.
- If `status: "error"` — attempt to fix (up to 2 retries):
  - `missing_package`: re-run renv setup
  - `syntax_error`: fix the R code
  - `object_not_found`: check variable names
- If still failing after retries, push anyway with a note about the error.

### 5. Push to GitHub

```bash
cd /output/app
git init
git checkout -b main
git add -A
git commit -m "feat: Teal app generated by TealFlow

Modules: {list from artifact}
{summary from artifact}"

git remote add origin "https://x-access-token:${GITHUB_TOKEN}@github.com/${GITHUB_REPO}.git"
git push -u origin main --force
```

If the repo already exists with content, use a feature branch:
```bash
git checkout -b tealflow/generated-app
git push -u origin tealflow/generated-app --force
```

### 6. Report result

Return:
```
## Deployed

**Repository**: {githubRepo}
**Branch**: main (or tealflow/generated-app)
**Modules**: {module list}
**Validation**: {ok or error details}
**Summary**: {app summary}
```

## Constraints

- Do not modify the app code from the artifact unless fixing a validation error.
- Use TealFlowMCP tools only for renv setup and validation — the code is already generated.
- If GITHUB_TOKEN is not available, write files and report success without pushing.
