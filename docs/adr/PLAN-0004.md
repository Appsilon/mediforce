# PLAN-0004 — Implementation plan for ADR-0004

Companion to [ADR-0004](./0004-scoped-data-access-authorization.md).
Implementation lands as **one PR** that rewrites the 10 GET handlers from
PR #450, introduces the wrapper layer, and ships Phase 2 mutations on
top. Approx 600 LOC total, mechanical.

> **Post-implementation note (deltas from spec):**
> - Base class is named `AuthorizedScope` (not `AuthorizedRepository<T>`)
>   and exposes only the two namespace primitives (`canSeeNamespace`,
>   `assertNamespaceWrite`). The entity-shaped helpers (`<T>`,
>   `namespaceOf`, `gate`, `filter`) the spec sketched are inlined at the
>   single direct-entity wrapper (`AuthorizedWorkflowRunRepository`).
>   Rule of three: re-extract when a second direct-entity wrapper appears.
> - Base class lives in `packages/platform-api/src/repositories/`
>   alongside the wrappers (not `platform-infra/firestore/`); the Postgres
>   swap is additive on the storage side and leaves the wrapper layer
>   unchanged.
> - Wrapper classes are exported without an `Impl` suffix; the parallel
>   `interface Authorized<Entity>Repository` declarations the spec
>   sketched were dropped (the class is the contract).

Domain terms used below — Caller, Workspace, Workflow Run
(`ProcessInstance`), Workflow Definition, Human Task, Cowork Session,
Agent Run, Audit Event, Handoff — defined in
[`CONTEXT.md`](../../CONTEXT.md).

## 2. Base class — Firestore era

ADR-0001 sec. 2.1 places the `AuthorizedRepository<TRow>` base class in
`packages/platform-infra/src/postgres/repositories/`. For ADR-0004's
Firestore-era introduction the equivalent lives in
`packages/platform-infra/src/firestore/authorized-repository.ts`. Same
shape; different default-read implementation (Firestore query builder
instead of Drizzle).

```ts
// packages/platform-infra/src/firestore/authorized-repository.ts
export abstract class AuthorizedRepository<T extends { workspace: string }> {
  constructor(protected readonly caller: CallerIdentity) {}

  /** Override when workspace is reached via a parent entity. */
  protected workspaceOf(entity: T): string | undefined {
    return entity.workspace;
  }

  protected canSee(entity: T | null): boolean {
    if (!entity) return false;
    if (this.caller.kind === 'apiKey') return true;
    const ws = this.workspaceOf(entity);
    return typeof ws === 'string' && this.caller.namespaces.has(ws);
  }

  protected gate(entity: T | null): T | null {
    return entity && this.canSee(entity) ? entity : null;
  }

  protected filter(entities: readonly T[]): T[] {
    if (this.caller.kind === 'apiKey') return [...entities];
    return entities.filter((e) => this.canSee(e));
  }
}
```

Post-Postgres swap, the base class moves to
`packages/platform-infra/src/postgres/repositories/base.ts` per ADR-0001
sec. 2.1 (reshaped per the patch landing on the ADR-0001 branch). The
wrapper interfaces in `platform-api` are unchanged across the swap.

## 3. Per-entity wrappers

### 3.1 Direct-namespace entities (~25 LOC each)

`WorkflowRun`, `WorkflowDefinition`, `AgentDefinition`, `ToolCatalogEntry`,
`OAuthProviderConfig`, `WorkspaceSecret`, `AgentOAuthToken`.

Example:

```ts
// packages/platform-api/src/repositories/authorized-workflow-run-repository.ts
export interface AuthorizedWorkflowRunRepository {
  getById(id: string): Promise<WorkflowRun | null>;
  list(opts: ListRunsOptions): Promise<WorkflowRun[]>;
  create(input: CreateWorkflowRunInput): Promise<WorkflowRun>;
  cancel(id: string): Promise<WorkflowRun>;
  resume(id: string): Promise<WorkflowRun>;
  // ... mirrors ProcessInstanceRepository surface area
}

export class FirestoreAuthorizedWorkflowRunRepository
  extends AuthorizedRepository<WorkflowRun>
  implements AuthorizedWorkflowRunRepository
{
  constructor(caller: CallerIdentity, private raw: ProcessInstanceRepository) {
    super(caller);
  }
  getById = async (id: string) => this.gate(await this.raw.getById(id));
  list = async (opts: ListRunsOptions) => this.filter(await this.raw.list(opts));
  create = async (input: CreateWorkflowRunInput) => {
    if (this.caller.kind === 'user' && !this.caller.namespaces.has(input.workspace)) {
      throw new ForbiddenError();
    }
    return this.raw.create(input);
  };
  cancel = async (id: string) => {
    if (!(await this.getById(id))) throw new NotFoundError();
    return this.raw.cancel(id);
  };
}
```

Note the canonical name `WorkflowRun` is introduced at the wrapper
interface; the underlying raw repo retains `ProcessInstanceRepository`
until ADR-0001's storage-level rename PR.

### 3.2 Indirect-namespace entities (~35 LOC each)

`HumanTask`, `CoworkSession`, `AgentRun`, `AuditEvent`, `Handoff` — all
reach their workspace via the parent `WorkflowRun`.

Pattern: constructor takes the parent's raw repository as a second
dependency; `getById` issues a parent lookup. List paths batch-dedupe
parent ids (same shape as today's `filterTasksByNamespace` in
[`packages/platform-ui/src/app/api/tasks/route.ts`](../../packages/platform-ui/src/app/api/tasks/route.ts)).

```ts
// packages/platform-api/src/repositories/authorized-human-task-repository.ts
export class FirestoreAuthorizedHumanTaskRepository
  extends AuthorizedRepository<HumanTask>
  implements AuthorizedHumanTaskRepository
{
  constructor(
    caller: CallerIdentity,
    private raw: HumanTaskRepository,
    private parentRuns: ProcessInstanceRepository,
  ) {
    super(caller);
  }

  getById = async (id: string) => {
    const task = await this.raw.getById(id);
    if (!task) return null;
    const run = await this.parentRuns.getById(task.processInstanceId);
    return this.caller.kind === 'apiKey' || (run && this.caller.namespaces.has(run.namespace ?? '')) ? task : null;
  };

  listByRole = async (role: string) => {
    const tasks = await this.raw.getByRole(role);
    if (this.caller.kind === 'apiKey') return tasks;
    const ids = [...new Set(tasks.map((t) => t.processInstanceId))];
    const runByIs = new Map<string, string | undefined>();
    await Promise.all(ids.map(async (id) => runByIs.set(id, (await this.parentRuns.getById(id))?.namespace)));
    return tasks.filter((t) => this.caller.namespaces.has(runByIs.get(t.processInstanceId) ?? ''));
  };

  claim = async (id: string, userId: string) => {
    if (!(await this.getById(id))) throw new NotFoundError();
    return this.raw.claim(id, userId);
  };
  // ... complete, cancel similar
}
```

## 5. Adapter changes

`packages/platform-ui/src/lib/route-adapter.ts` swaps the handler
signature contract from `(input, caller)` (PR #450) to `(input, scope)`.

```ts
// Before (PR #450)
export type RouteHandler<Input, Output> = (input: Input, caller: CallerIdentity) => Promise<Output>;

// After (this PR)
export type RouteHandler<Input, Output> = (input: Input, scope: CallerScope) => Promise<Output>;
```

The adapter resolves the caller (unchanged from PR #450), then builds a
`CallerScope` from the global `PlatformServices`, then invokes the
handler with `(parsedInput, scope)`. Route files no longer need to
destructure `getPlatformServices()` — the adapter does it once.

Per-request cost: one `CallerScope` construction. Allocates 13 small
wrapper instances. Negligible.

Streaming endpoints (Phase 3) extend the handler return type to
`Output | StreamResponse`. Adapter returns `NextResponse.json(...)` or
`new NextResponse(stream, { headers })` based on the shape.

## 8. Static guard

```ts
// packages/platform-api/src/handlers/__tests__/no-raw-repo-imports.test.ts
describe('platform-api handlers may not import raw repositories', () => {
  it('every file under handlers/ imports only from @mediforce/platform-api', () => {
    const banned = [
      /from\s+['"]@mediforce\/platform-core\/interfaces/,
      /from\s+['"]@mediforce\/platform-infra/,
    ];
    const offenders: string[] = [];
    for (const file of listHandlerFiles(HANDLERS_ROOT)) {
      const src = readFileSync(file, 'utf8');
      if (banned.some((re) => re.test(src))) offenders.push(file);
    }
    expect(offenders).toEqual([]);
  });
});
```

Replaces the previous `auth-coverage.test.ts` (PR #450). Strictly
structural — no regex on auth helper names, no `@public-handler`
escape hatch needed because raw-repo access is the only way to bypass
the wrapper.

## 9. Test strategy

- **L2 (handler tests)** — every handler tested with an in-memory
  `CallerScope` built from in-memory repos (already in
  `platform-core/src/testing/`). Existing PR #450 L2 tests migrate by
  swapping the test setup builder; assertion shape unchanged.
- **L3 (route adapter + API E2E)** — PR #450's `route-adapter.test.ts`
  updates one type. The 6 multi-namespace journey files
  (`tasks-get.journey.ts`, `processes-detail.journey.ts`, etc.) stay
  verbatim; they assert HTTP shapes and 404 anti-enumeration, which is
  unchanged.
- **L1 (wrapper unit tests)** — each `Authorized<Entity>Repository`
  gets one test file covering: in-scope return, out-of-scope null/404,
  apiKey bypass, soft-delete filter (for entities that have it).
- **`no-raw-repo-imports.test.ts`** — new structural test.

## 11. Roll-out / roll-back

- **Roll-out.** Single PR merge. The wrapper layer is internal; no
  external API contract changes (HTTP request/response shapes are
  unchanged — same 404 anti-enum semantics from PR #450).
- **Roll-back.** Revert the merge. All handler files retain a tight
  diff against PR #450's shape, so revert is a clean operation. The
  wrapper layer is purely additive infrastructure; reverting removes
  the imports and the static guard.
