How we think about validated AI

Validate the process, not the AI.

You can't validate an AI agent the way you validate software. So we validate everything around it — the checks, the enforcement, the review, and the proof that it all actually happened.

Four categories of mechanisms make this work.

Test

Run it a thousand times. Measure what you get.

Statistical testing for non-deterministic systems.

Agent benchmark suites

Define test cases. Run an agent 50 or 500 times. Measure consistency, quality, and scope compliance.

Workflow evals

Run entire multi-step workflows against scenarios. Measure final output quality end-to-end.

Deterministic testing

Routing, permissions, transforms — tested traditionally. The engine is fully deterministic.

Drift monitoring

Benchmarks re-run on schedule. Quality degrades? You know before production does.

Watch

Monitor the agents. Monitor the humans.

Testing catches problems before deployment. Monitoring catches them in production.

Judge agents

An agent that evaluates other agents. Every output or spot-check a %. Scores, flags, escalates to humans.

Reviews 100% of safety narratives. Spot-checks 10% of classifications. Flags below threshold.

Live activity

Real-time dashboard: running, stuck, waiting for review. Task queues per role. Instant updates.

Control

Define the rules. The platform enforces them.

Not guidelines. Enforcement. Configured rules cannot be bypassed.

Autonomy levels

Each step: who decides? You choose per step.

Autonomy Configuration
L0 Human only No agent involvement
L1 Agent assists Human decides
L2 Agent annotates Output collected, not applied
L3 Agent proposes Pauses until human approves
L4 Agent decides Applied automatically

Producer ≠ reviewer

The agent that produces output cannot review it. Separation of duties.

Per-step access control

Each step specifies allowed roles. Wrong role? Blocked and logged.

Data scoping

Agents access only data for their step. Study A can't see Study B.

Confidence thresholds

Below threshold? Auto-escalation to human regardless of autonomy.

Prove

Show exactly what happened.

Structured records with snapshots and tamper-evident timestamps.

Audit trail

Who, what, when, why, before/after. Append-only.

Decision reconstruction

Input, config, raw output, human changes, final version. Don't re-run — reconstruct.

Qualification reports

Pass/fail, consistency, scope tests, drift history. Auditor-ready.

Not everything needs maximum rigor.

Output goes into FDA submission? Full stack, L1–L3 only. Supply chain logistics? Control + Prove, L3–L4 fine. Rigor proportional to risk — from ICH E6(R3).

Submission-adjacent — all four pillars, full rigor, L1–L3.
Regulated operations — Test + Control + Prove, L2–L3.
Operational — Control + Prove recommended, L3–L4 fine.

This is a starting point, not a finished standard.

If you see gaps — especially if you think we're wrong — that's the conversation we want.
