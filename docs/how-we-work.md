# How We Work

## Building in Public

Mediforce is being built in the open. Not because we have everything figured out, but because we believe the standard for human-agent collaboration in pharma should be created together — not behind closed doors.

What this means in practice:
- Architecture decisions, trade-offs, and design rationale are shared publicly
- We show the process of building, not just the results
- We welcome input at every stage — especially from people who know these processes from the inside

## Our Approach

### Bottom-Up, Not Top-Down

We don't start with a grand abstraction and hope it fits reality. We start with a real process (clinical monitoring), build a working application, and extract the platform patterns as they emerge.

Every abstraction in Mediforce earned its place by being needed in more than one concrete process. If a pattern only appears once, it stays in the application.

### Code-First

Mediforce is a developer platform, not a drag-and-drop builder. We believe that code is the most precise, version-controllable, and AI-friendly way to define processes. Good APIs, clear documentation, and TypeScript throughout.

That said, AI coding assistants (like Claude Code, which we use extensively in development) make code-first accessible to people who think in processes, not in syntax.

### Real Processes, Not Demos

Every feature we build is validated against real pharma processes with real complexity. Clinical monitoring involves parallel steps, review loops, multiple agents with different autonomy levels, compliance gates, and complex data models. If the platform handles that, simpler processes are easy.

## The Team

Mediforce is built by [Appsilon](https://appsilon.com) — a company that has been building data solutions for life sciences since 2013. We've worked with some of the largest pharma companies in the world, building production software for clinical operations, biostatistics, and data science teams.

We started building a medical monitoring application and realized: every pharma company will need this infrastructure. It shouldn't be reinvented for every project. It should be a standard.
